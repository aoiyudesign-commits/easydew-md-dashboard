# EASYDEW MD 그로스 대시보드

디엔코스메틱스 뷰티MD 채용 지원용 AI 기반 업무 프로토타입입니다. 서비스기획자 → MD 직무 전환을 준비하며,
JD의 핵심 업무(직영몰 손익/ROAS 관리, 고객 인사이트 발굴, 광고 소재 테스트, 경쟁사 대응)를
AI 도구로 구현해봤습니다.

## 실행 방법

빌드 과정이나 서버가 필요 없는 순수 HTML/CSS/JS 프로젝트입니다.

- 가장 간단하게는 `index.html`을 브라우저로 더블클릭해서 열면 됩니다.
- VSCode에서 작업할 경우 **Live Server** 확장을 설치한 뒤 `index.html`에서 우클릭 → "Open with Live Server"를 사용하면 저장할 때마다 자동 새로고침됩니다.

## 파일 구조

```
이지듀 프로젝트/
├── index.html   # 페이지 구조 (사이드바 메뉴 3개: 경쟁사 가격 / 리뷰 인사이트 / ROAS 시뮬레이터)
├── style.css    # 전체 스타일 (CSS 변수 기반 디자인 시스템)
├── script.js    # 데이터(products/competitors/reviewInsights) + 메뉴별 로직 + Chart.js 렌더링
├── scripts/
│   ├── crawl_reviews.py         # easydew.co.kr 리뷰 게시판(board_no=4) 크롤러 — VOC 모니터링 피드용
│   └── crawl_crema_reviews.mjs  # 상품 상세페이지 리뷰 위젯(Cre.ma) API 크롤러 — 리뷰 톤·키워드·세그먼트 분석용
└── 이지듀_MD_대시보드_설명자료.pdf   # 자소서/포트폴리오 첨부용 1페이지 요약
```

외부 의존성은 CDN에서 불러오는 [Chart.js 4.4.0](https://www.chartjs.org/) 하나뿐입니다 (`index.html` 상단 `<script>` 태그).

## 데이터 출처 및 주의사항

- 이지듀 제품명/가격/리뷰수: easydew.co.kr 공식몰 실데이터 (2026.07 조사)
- 경쟁사(넘버즈인·구달·AXIS-Y) 가격: 올리브영·각 브랜드 공식몰 공개 정보 (2026.07 조사)
- 리뷰 데이터는 **두 개의 실제 소스**를 함께 씁니다.
  - **VOC 모니터링 피드**: [상품 사용후기 게시판](https://easydew.co.kr/board/product/list.html?board_no=4)(구형 Cafe24 게시판)을 크롤링. `scripts/crawl_reviews.py` — 목록 페이지를 순회해 리뷰 URL을 모으고, 상세페이지의 JSON-LD(`articleBody`)에서 원문을 추출.
  - **리뷰 톤·키워드·고객 세그먼트 분석**: 실제 상품 상세페이지에 붙어있는 리뷰 위젯([Cre.ma](https://www.cre.ma/))의 API에서 직접 수집. `scripts/crawl_crema_reviews.mjs` — 공식 문서화된 공개 API는 아니고 위젯이 브라우저에서 호출하는 엔드포인트를 그대로 재사용. 리뷰마다 고객이 직접 선택한 피부타입·피부고민·연령대·구매여부 태그, 그리고 벤더가 자체 생성한 AI 한줄요약·키워드(`ai_summary`)까지 함께 내려온다는 걸 발견해서, 자체 정규식 키워드 추출 대신 이 구조화 데이터를 그대로 활용했습니다. 제품별 표본 60건 기준(전수조사 아님, 2026.07 수집). 전체 리뷰수·평점(`metaCount`/`metaScore`)은 위젯이 표시하는 실제 값입니다.
- 원가율·물류비·손익분기·일별 가격 변동 로그 등 **비공개 재무/운영 데이터는 업계 평균 가정치**이며, `script.js` 상단 주석과 대시보드 내 "제작 노트"에 명시되어 있습니다. 실제 데이터로 교체하면 바로 운영 가능한 구조입니다.

## 진행 현황

- ✅ 리뷰 인사이트(② 탭): 시연용 예시 문구 → **Cre.ma 리뷰 위젯 API 기반 실데이터 4개 제품**으로 교체 (기미앰플 실속 세트·기미 앰플 쿠션 세트 1+1·기미 집중 패치·기미 톤업 썬세럼). AI 종합 요약(벤더 제공), 키워드(벤더 AI 태그 집계), 실제 리뷰 원문 발췌까지 전부 실데이터
- ✅ **고객 세그먼트 분석**(② 탭): 리뷰 작성 시 고객이 직접 선택한 연령대·구매여부·피부과 관리 경험·피부고민을 도넛/바 차트로 시각화. 특정 제품의 리뷰 폼에서 "피부고민" 응답에 피부타입 값이 혼입되는 데이터 품질 이슈를 발견해 정상 응답만 집계하고 그 사실 자체를 인사이트로 노출
- ✅ **VOC 모니터링 피드**(② 탭 상단): 특정 제품에 한정하지 않고 리뷰 게시판(board_no=4) 전체 카테고리에서 수집한 실제 리뷰 36건을 "신규 리뷰 인박스" 형태로 표시. `classifyCategory()`/`classifyIssues()`가 제품명·리뷰 원문을 규칙 기반으로 자동 분류(카테고리 태그·이슈 태그)하고, 3점 이하는 "CS 대응 필요"로 자동 플래그. 카테고리/별점 필터 제공. 실 서비스에서는 스케줄러가 신규 게시물만 수집해 이 배열에 append하면 됨
- ✅ 가격 변동 알림(③ 탭): `checkPriceAlerts(thresholdPct)`로 함수화, 화면에서 임계치(%)를 조정하면 실시간 재계산. 실 연동 시 스크래핑 스케줄러(cron)가 `competitors`를 갱신한 뒤 이 함수만 호출하면 됨
- ✅ 데이터 소스 연동 지점: `products`/`competitors`를 `loadProducts()` / `loadCompetitors()` 비동기 함수로 감쌈. 실 연동 시 두 함수 내부만 카페24 Admin API·엑셀 export 파서로 교체하면 나머지 로직은 그대로 재사용 가능
